## svc_es 9200端口不可达 SOP

### 告警识别

收到 ES 集群 9200 端口不可达告警后，确认目标节点：
- **master-1**：10.33.17.100
- **master-2**：10.33.17.101

本 SOP 适用于上述两台 ES Master 节点的 9200 端口故障场景。

### 前置条件

- **权限**：aiagent 用户，本机直连（无需跳板机）
  ```bash
  # ES 主机在 10.33.17.x 网段，本机可直接 SSH
  ssh aiagent@10.33.17.100
  ssh aiagent@10.33.17.101
  ```
- **影响范围**：9200 端口不可达会导致 ES HTTP API 不可用，影响 Kibana、Logstash 及所有依赖 ES 的服务
- **目标节点**：10.33.17.100（master-1，含 Kibana）或 10.33.17.101（master-2）
- **重启说明**：`systemctl restart elasticsearch` 执行后无输出返回，属正常现象，等待约 30-60 秒 ES 恢复即可

### 操作步骤

**Step 1 — 登录目标节点**

```bash
# 本机直连（无需跳板机）
ssh aiagent@10.33.17.100

# 或
ssh aiagent@10.33.17.101
```

**Step 2 — 检查 9200 端口状态**

```bash
netstat -ntlp | grep 9200
```

正常输出示例：
```
tcp6       0      0 :::9200                 :::*                    LISTEN      39724/java
```

如果无输出，说明 9200 端口未监听，继续下一步。

**Step 3 — 重启 Elasticsearch**

```bash
sudo systemctl restart elasticsearch
```

> ⚠️ 此命令执行后无回显，属正常行为。等待约 30-60 秒后验证。

**Step 4 — 验证恢复**

```bash
# 等待 30 秒
sleep 30

# 重新检查端口
netstat -ntlp | grep 9200

# 检查 ES 进程
ps aux | grep elasticsearch | grep -v grep

# 检查集群健康（可选）
curl -s http://localhost:9200/_cluster/health | python3 -m json.tool
```

### 验证

- 检查 9200 端口：`netstat -ntlp | grep 9200`（应有 LISTEN）
- 检查 ES 进程：`ps aux | grep elasticsearch | grep -v grep`
- 检查集群健康：`curl -s http://localhost:9200/_cluster/health`（status 应为 green 或 yellow）
- 检查 ES 服务状态：`sudo systemctl status elasticsearch`

### 回滚

- 重启操作可通过 `sudo systemctl stop elasticsearch && sudo systemctl start elasticsearch` 再次执行
- 如果重启后仍无 9200 端口，检查 ES 日志：`tail -100 /data/elasticsearch-8.3.3/logs/elasticsearch.log`
- 如果 master-1（含 Kibana）故障，Kibana 服务也需确认：`sudo systemctl status kibana`

### 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| restart 后 9200 仍不可达 | JVM OOM、磁盘满、配置错误 | 检查 ES 日志 `tail -100 /data/elasticsearch-8.3.3/logs/elasticsearch.log` |
| restart 后端口恢复但集群状态 red | 分片损坏或节点数据不一致 | `curl localhost:9200/_cat/shards?v` 查看未分配分片 |
| 两台 Master 同时告警 | 集群级故障（网络分区、磁盘满等） | 优先级 P0，需人工介入排查根因，不要同时重启两台 |
| 重启过程 Kibana 不可用 | master-1 上 Kibana 依赖 ES 9200 | 等待 ES 恢复后 Kibana 自动恢复 |
